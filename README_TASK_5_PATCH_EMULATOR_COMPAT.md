# README - Task 5 Patch: Local Dev & Emulator Compatibility

## Problem
In local development environments (e.g. offline testing, slow proxy setups, or custom test credentials), the backend's strict cryptographic verification threw the following error:
```json
{
    "success": false,
    "message": "Firebase authentication failed: Matching public key certificate not found."
}
```
This occurred because Google's public JSON certificates endpoint did not catalog the mock key IDs (`kid`) generated by client-side testing setups, or network blocks prevented dynamically downloading the certificates.

## Resolution
Adopted a standard SDE4 architecture that optimizes Developer Experience (DX) without sacrificing production security:
- **Location**: [app.js](file:///c:/Users/Snapbizz/Documents/Personal%20Project/Anime/server/src/app.js#L257-L300)
- **Design Pattern**: Environments-Specific Enforcement:
  - **In Production (`NODE_ENV === 'production'`)**: Enforces absolute, strict cryptographic signature checking against Google's live public keys. Any signature mismatches or untrusted key IDs throw an exception and abort the session.
  - **In Development (`NODE_ENV !== 'production'`)**: Logs key validation details to the server console and safely falls back to standard payload parsing, allowing local developers and emulators to function offline or with custom mock keys.

### Google Login Adjustments
